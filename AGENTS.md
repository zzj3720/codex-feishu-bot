# AGENTS.md

This repository is designed for a Codex-operated bootstrap flow.

## Primary Goal

Run the one-click Feishu bot creation flow with as little manual work as possible. The intended operator is another Codex session running on the user's machine.

## Required Execution Order

1. Read `README.md`.
2. Read `docs/codex-bootstrap-playbook.md`.
3. Read `docs/open-source-scope.md`.
4. Run `npx -y lark-op-cli@latest create-bot`.
5. Stream the command output continuously.
6. If login is required, surface the ASCII QR code or prompt back to the user immediately.
7. Summarize the creation result, identifiers, generated files, and next steps.

## Creation Rules

- Do not open a browser.
- Do not use `agent-browser`.
- Do not rely on Chrome DevTools Protocol automation.
- Create the bot by running `npx -y lark-op-cli@latest create-bot` directly.
- Keep reading stdout/stderr until the command finishes or clearly blocks on user action.

## What Still Requires the User

- Scanning the Feishu login QR code if the script requires login.
- Logging into OpenAI/Codex if the local `~/.codex` state is missing.
- Approving Feishu-side prompts if the organization requires them.

Only stop for those checkpoints. Do not route the user through browser-based console operations.

## Execution Rules

- Do not run `pnpm chrome:debug`.
- Do not install or invoke `agent-browser`.
- Do not continue into Docker deployment as part of this prompt path.
- Never commit `.env.real` or local state generated by the creation flow.

## Success Criteria

- `npx -y lark-op-cli@latest create-bot` has been executed.
- Any login QR code or prompt has been surfaced to the user in time.
- The user receives the creation result, key identifiers, generated files, and next steps.
